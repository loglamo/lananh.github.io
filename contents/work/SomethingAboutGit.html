Something about Git

Git

This is a distributed sourcecode management tool. It means that it helps you manage, update, store your sourcecode

Basics of Git

  • Git uses "Repository(repo)" to store all about your source code (your project): code, modify management,...Two types of repo: Local and Remote. You can store your project in local. If you want to store for safety or share others to joint work, you can push your project to Remote repo.
  • Some websites for Git: Github , GitLab,.etc.

Work with Github

  1. Install: The first step you have to do is installing git in your computer so that you can use Git commands in your computer. You also have to create an account in github website so that it is like your remote repo.
  2. Basics of Github workflow: Clone Project(clone the project in remote to your computer so that you can modify the project as you want) -> Modify Project as you want -> Git add (add things you modified into temporary memory) -> Git commit (Changes to your local repository) -> Git push (push your changes to remote repository).
  3. More details at Github workflow
    4. you can find the architecture of github there, basic commands of git.
  4. Git have two main types of users: Master, and distributors. Master is the man who created the repository in the remote. distributors are the men who wanted to modify something in the project of the master.
  5. Basically, you should remember: git pull- git add- git commit- git push (update modifies- add your modifies to the stage- push to remote repository)
  6. Other commands are git fork, git pull request, git merge,...This Cheat Sheet may help you.
